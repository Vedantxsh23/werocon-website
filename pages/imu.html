Module 01 / Sensing — Raspberry OS Setup

IMU — Inertial Measurement

Step-by-step setup for the prosthesis's sensing core: flash Raspberry Pi OS onto the Pi 5, connect over SSH, then wire and read a 6-axis MPU6250 accelerometer + gyroscope over I2C to capture gait-phase data.

PLATFORM Raspberry Pi 5
SENSOR MPU6250 (6-axis IMU)
BUS I2C — /dev/i2c-1
DEFAULT ADDR 0x68

Step 1 — Download Raspberry Pi Imager

On your laptop, download the imager from:

https://www.raspberrypi.com/software/

Install and open it.

Step 2 — Insert microSD Card

Plug your microSD card into the card reader, then plug it into your laptop.

Step 3 — Select Device, OS & Storage

In Raspberry Pi Imager, set the following:

FieldWhat to Choose
Choose DeviceRaspberry Pi 5
Choose OSRaspberry Pi OS (other) → Raspberry Pi OS Lite (64-bit)
Choose StorageYour microSD card

Click Next.

Step 4 — Edit Settings (Important)

When asked to apply custom settings → click Edit Settings.

General tab

Hostname:       raspberrypi
Username:       pi
Password:       [choose a password]
WiFi SSID:      [your WiFi name]
WiFi Password:  [your WiFi password]

Services tab

[x] Enable SSH   ← Must turn this ON

Click Save → Yes → Yes.

⏳ Flashing takes about 5–10 minutes.

Step 5 — Boot the Pi

Eject the microSD card from your laptop, insert it into the bottom slot of the Raspberry Pi 5, plug in the power cable, then wait 60 seconds.

Step 6 — Connect via SSH

Open Terminal (Mac/Linux) or Command Prompt (Windows):

ssh pi@raspberrypi.local

Type your password when asked. You should see:

pi@raspberrypi:~ $
If it doesn't connect, find your Pi's IP from your WiFi router and try: 
ssh pi@192.168.x.x

Step 7 — Update the System

Once connected, run:

sudo apt update && sudo apt upgrade -y

Done — your Raspberry Pi 5 is ready to use. ✅

Quick Summary — Flash to SSH

MPU6250 — Wiring & Setup

Connect and read accelerometer + gyroscope data from the MPU6250 using I2C on the Raspberry Pi 5.

Step 8 — Enable I2C on Raspberry Pi 5

The MPU6250 communicates via I2C, which is disabled by default. Enable it:

sudo raspi-config

Navigate using arrow keys:

Interface Options → I2C → Yes → OK → Finish

Then reboot:

sudo reboot

SSH back in after reboot:

ssh pi@raspberrypi.local

Verify I2C is active:

ls /dev/i2c*
# Should show: /dev/i2c-1

Step 9 — Wire the MPU6250 to Raspberry Pi 5

Connect using 4 jumper wires only:

MPU6250 PinRaspberry Pi 5 PinDescription
VCCPin 1 — 3.3VPower
GNDPin 6 — GNDGround
SDAPin 3 — GPIO 2I2C Data
SCLPin 5 — GPIO 3I2C Clock
⚠ Always use 3.3V for VCC — never 5V. The MPU6250 is a 3.3V sensor and 5V will damage it.

Pi 5 GPIO Diagram (relevant pins)

1 3.3V ← VCC (MPU6250 power)
2 5V not used
3 GPIO 2 (SDA) ← SDA (I2C Data)
4 5V not used
5 GPIO 3 (SCL) ← SCL (I2C Clock)
6 GND ← GND (Ground)

AD0 pin — I2C address

AD0I2C Address
GND (or unconnected)0x68 ← default
3.3V0x69

Leave AD0 unconnected to use the default address 0x68.

Install I2C tools

sudo apt install -y i2c-tools python3-smbus

Create a virtual environment

python3 -m venv .venv
source .venv/bin/activate
You will see (.venv) at the start of your terminal — this means it is active.

Install Python libraries

Step 12 — Read IMU Data

Create a Python script:

nano read_mpu.py

Paste this code:

import smbus2
import time

MPU6250_ADDR = 0x68   # Change to 0x69 if AD0 is connected to 3.3V
PWR_MGMT_1   = 0x6B
ACCEL_XOUT_H = 0x3B
GYRO_XOUT_H  = 0x43
TEMP_OUT_H   = 0x41

bus = smbus2.SMBus(1)

# Wake up the MPU6250
bus.write_byte_data(MPU6250_ADDR, PWR_MGMT_1, 0x00)
time.sleep(0.1)

def read_word(reg):
    high  = bus.read_byte_data(MPU6250_ADDR, reg)
    low   = bus.read_byte_data(MPU6250_ADDR, reg + 1)
    value = (high << 8) | low
    if value >= 0x8000:
        value -= 65536
    return value

def get_accel():
    ax = read_word(ACCEL_XOUT_H)     / 16384.0
    ay = read_word(ACCEL_XOUT_H + 2) / 16384.0
    az = read_word(ACCEL_XOUT_H + 4) / 16384.0
    return ax, ay, az

def get_gyro():
    gx = read_word(GYRO_XOUT_H)     / 131.0
    gy = read_word(GYRO_XOUT_H + 2) / 131.0
    gz = read_word(GYRO_XOUT_H + 4) / 131.0
    return gx, gy, gz

def get_temp():
    raw = read_word(TEMP_OUT_H)
    return (raw / 340.0) + 36.53

print("MPU6250 Live Data — Press Ctrl+C to stop\n")

try:
    while True:
        ax, ay, az = get_accel()
        gx, gy, gz = get_gyro()
        temp       = get_temp()

        print(f"Accel (g)    X:{ax:+.3f}  Y:{ay:+.3f}  Z:{az:+.3f}")
        print(f"Gyro (°/s)   X:{gx:+.3f}  Y:{gy:+.3f}  Z:{gz:+.3f}")
        print(f"Temp (°C)    {temp:.2f}")
        print("-" * 45)
        time.sleep(0.5)

except KeyboardInterrupt:
    print("Stopped.")

Save with Ctrl+X → Y → Enter, then run it:

python3 read_mpu.py

Expected output

Gyro (°/s)   X:+0.213  Y:-0.152  Z:+0.031
Temp (°C)    28.54

Troubleshooting

ProblemFix
i2cdetect shows nothingRecheck wiring — SDA→Pin3, SCL→Pin5, VCC→3.3V, GND→GND
Permission denied on I2CRun sudo usermod -aG i2c pi then log out and back in
ModuleNotFoundError: smbus2Run source .venv/bin/activate then pip install smbus2
All readings are 0Reinstall: pip uninstall smbus2 && pip install smbus2
Address shows 69 not 68Change MPU6250_ADDR = 0x69 in the script
